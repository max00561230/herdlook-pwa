Animal Reference Photo Guide

Use this guide when creating a new animal profile. More good reference photos improve camera-assisted lookup results.

Minimum beta requirement: Add at least 3 clear photos before relying on camera lookup: front face, left side, and right side. Recommended: 6 to 8 photos per animal.

Cattle: Recommended 6 Photos

1 Front face
2 Left side full body
3 Right side full body
4 Rear view
5 Ear tag / brand close-up
6 Unique marking or muzzle/nose pattern close-up

Horses: Recommended 7 Photos

1 Front face
2 Left side full body
3 Right side full body
4 Rear view
5 Face / blaze close-up
6 Legs / socks / markings close-up
7 Brand, scar, microchip proof, or unique marking close-up

Photo Quality Rules

Use good daylight or even barn lighting.
Keep the animal filling most of the frame.
Avoid blurry photos and fast motion.
Avoid heavy shadows and glare.
Avoid fences, gates, people, or other animals blocking the subject.
Take side photos with the full body visible.
Capture unique markings, tags, brands, scars, blaze, socks, muzzle/nose patterns, or coat patterns.
Update photos when the animal changes: seasonal coat, weight change, injury, new tag, or major grooming change.
App wording: HerdLook does not identify animals with certainty. It suggests possible matches based on saved reference photos. Always confirm before opening or updating records.

How to Use HerdLook Beta

1. Set up owner and farm/ranch info

Open Owner/Farm and add the owner name, farm/ranch name, contact info, vet info, and notes. This helps make reports more complete.

2. Add animal profiles

Create one profile per animal. Add animal ID, name, species, breed, tag/RFID/microchip, location, and markings.

3. Add good reference photos

Use the Photo Guide. Minimum beta set: front face, left side, and right side. Best set: 6 to 8 clear photos covering face, both sides, rear, tag/brand/ID, and unique markings.

4. Use Camera Lookup

Take or upload a photo. HerdLook will suggest possible matches. Always confirm manually before opening or changing records.

5. Add records

Log health, weight, breeding, movement, notes, and sale/transfer events. Add follow-up dates when needed.

6. Store documents

Use the Documents page for vet papers, registration papers, receipts, test results, or bills of sale.

7. Add map/locations

Add pastures, barns, pens, gates, and water points. Use location names consistently on animal profiles.

8. Export backups regularly

Because HerdLook is local-only by default, use Data/Privacy or Settings to export a full backup file. Save that file somewhere safe.

9. Turn on a PIN lock

Go to Settings and set a PIN. This protects casual access on the device, but it is not a replacement for full encryption.

Local-Only Data Storage

HerdLook is designed for users who prefer not to use cloud storage for farm, ranch, herd, and document records.

No cloud by default: This beta does not use Firebase, Supabase, server accounts, cloud sync, or external databases. Data stays in this browser/device unless the user exports it.
What is stored locally?

Owner/farm info, animal profiles, records, reference photos, stored documents, map/locations, and PIN lock settings.

What can delete local data?

Clearing browser/site data, using private browsing, device loss, damaged device storage, or uninstalling the app/browser can remove records.

How should users protect records?

Use Export Full Backup regularly and save the backup file somewhere safe, such as an external drive, secure local folder, or encrypted storage device.

Production recommendation

For a sold version, upgrade storage to IndexedDB and offer optional encrypted backup files. Keep cloud sync optional, not required.

Claw Build Direction

Use this beta version inside the PWA that Claw has already started. Keep the beta local-only, no cloud storage, and collect user reviews before building the full production version.

Beta goal: prove the workflow: animal profile → guided reference photos → camera-assisted suggestion → manual confirmation → records. Do not build a full AI livestock identity platform until beta feedback confirms the workflow.

Recommended next build steps

  1. Add IndexedDB storage for larger local photo and document libraries.
  2. Keep the main product local-first with no required cloud account.
  3. Add RFID/tag scanning support.
  4. Replace beta photo-similarity with a stronger livestock recognition model.
  5. Add audit history so record changes can be tracked.
  6. Add encrypted local document storage and optional encrypted backup files.
  7. Add GPS map integration, pasture boundaries, and offline map tiles without requiring cloud sync.
  8. Only add cloud sync later as an optional user-controlled feature, not the default.
