Photo Capture Best Practices

Minimum beta requirement: 3 clear photos: front face, left side, right side. Recommended: 6-8 photos per animal.

Cattle: 6 photos

  1. Front face
  2. Left side full body
  3. Right side full body
  4. Rear view
  5. Ear tag / brand close-up
  6. Unique marking or muzzle/nose close-up

Horses: 7 photos

  1. Front face
  2. Left side full body
  3. Right side full body
  4. Rear view
  5. Face / blaze close-up
  6. Legs / socks / markings
  7. Brand, scar, or unique marking

Quality rules

  • Use good daylight or even barn lighting.
  • Keep the animal filling most of the frame.
  • Avoid blur, fences, shadows, people, and other animals blocking the view.
  • Update photos after seasonal coat changes, new tags, injury, grooming, or weight change.

Local-Only Data

HerdLook stores data in this browser/device by default. No Firebase, Supabase, cloud database, server accounts, or automatic cloud sync.

Export backups regularly. Clearing browser data can delete local HerdLook records.

Backup / Restore

Import safety rule: Importing a backup will not overwrite existing data. To import a backup, first export a safety backup, then clear/delete the old local data from Settings.
Migration fix: Old HerdLook backups are normalized during import so animal category cards, Animal Photos, Records, Documents, Locations, owner/farm info, dam/sire fields, breed, and location fields display in the updated layout.
iPhone transfer rule: Full backups with photos/documents can be too large for iPhone import. For iPhone, export a Mobile Transfer Backup from desktop, then import that smaller file on iPhone.
Import status: Choose a backup file. On iPhone, use a Mobile Transfer Backup.
